# Baileys Feature Backlog

Recommended next features after the hardening pass, based on the Baileys 6.x docs and the runtime now being prepared for richer event handling:

## Core Messaging and Groups

- Message edit awareness
- Message delete awareness
- Reaction tracking
- Read/played receipt summaries
- Richer group metadata and participant tools
- Join request review where supported

## Status and Presence

- Status saver flow using captured `status@broadcast` messages
- Presence-aware user experience where useful
- Better history-sync-aware profile and search features

## Privacy

- Block and unblock commands
- Privacy settings inspection
- Last seen / online / read receipts / status privacy commands

## Business and Commerce

- Business profile lookup
- Product and catalog browsing for business accounts
- Order detail helpers when relevant
- Chat/message labels if practical for the linked account

## Bot Runtime Quality

- Optional pairing code login flow
- Richer retry support based on stored messages
- Better operator observability around message lifecycle events
